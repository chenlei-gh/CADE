# CATIA Dynamic Detection System

## Overview

CADE now features a **fully dynamic CATIA detection system** that eliminates all hardcoded paths and version numbers. The system automatically scans all available drives and paths to find CATIA installations, supporting any version from B20 to B99+ and R2018+.

## Architecture

### Components

```
tools/catia_detector.py          # Core detection engine
  ├─ CATIAInstallation          # Represents a detected CATIA
  └─ CATIADetector               # Scans drives and validates installations

tools/setup_environment.py       # Workspace configuration
  ├─ detect_catia_installations_interactive()
  ├─ select_catia_installation()
  └─ setup_workspace_environment()

skills/env.py                    # Build/runtime environment
  └─ _auto_detect()              # Uses detector for env initialization

skills/cade.py                   # CLI integration
  └─ cmd_setup()                 # User-facing command
```

### Detection Strategy

1. **Drive Scanning**: Dynamically scans C-Z for available drives
2. **Path Patterns**: Searches common installation patterns:
   - `<Drive>:\Program Files\Dassault Systemes\`
   - `<Drive>:\Program Files (x86)\Dassault Systemes\`
   - `<Drive>:\CATIA\`
   - `<Drive>:\DS\CATIA\`
   - `<Drive>:\Dassault Systemes\`
   - `<Drive>:\CAA\CATIA\`

3. **Version Recognition**: Regex pattern `^([BR])(\d{2,3})$`
   - Matches: B28, B30, R2018, R2020, etc.
   - Extracts version and release info

4. **Validation**: Checks for CATIA markers:
   - `CNext/` directory
   - `intel_a/` or `win_b64/` architecture folders
   - `code/bin/` structure

5. **Sorting**: Returns newest version first (B30 > B28, B versions > R versions)

## Usage

### Command Line

#### Detect CATIA installations
```bash
cade setup --detect
```

Output:
```
🔍 Scanning for CATIA installations...
✅ Found 2 CATIA installation(s)

B30:
  Path: D:\CATIA\B30
  Release: V5R30
  CNEXT.exe: D:\CATIA\B30\win_b64\code\bin\CNEXT.exe
  Code/Bin: D:\CATIA\B30\win_b64\code\bin

B28:
  Path: C:\Program Files\Dassault Systemes\B28
  Release: V5R28
  CNEXT.exe: C:\...\B28\win_b64\code\bin\CNEXT.exe
  Code/Bin: C:\...\B28\win_b64\code\bin
```

#### Configure workspace (interactive)
```bash
cd my_workspace
cade setup
```

If multiple versions found:
```
📋 Detected CATIA installations:
  [1] B30 - D:\CATIA\B30
      (Code/Bin: D:\CATIA\B30\win_b64\code\bin)
  [2] B28 - C:\Program Files\Dassault Systemes\B28
      (Code/Bin: C:\...\B28\win_b64\code\bin)

Select CATIA version [1-2] (default: 1): 1
✅ Workspace environment configured with CATIA B30
```

#### Non-interactive mode
```bash
cade setup --catia-root "D:\CATIA\B30"
```

### Python API

```python
from tools.catia_detector import detect_catia_installations, CATIAInstallation

# Detect all installations
installations = detect_catia_installations(verbose=True)

for inst in installations:
    print(f"Version: {inst.version}")
    print(f"Path: {inst.root_path}")
    print(f"Release: {inst.release}")
    print(f"Code/Bin: {inst.get_code_bin_path()}")
```

```python
from tools.setup_environment import setup_workspace_environment

# Configure workspace with auto-detection
result = setup_workspace_environment(
    workspace=Path("./my_workspace"),
    interactive=True  # Allow user selection
)

if result["status"] == "ok":
    config = result["config"]
    print(f"CATIA Root: {config['catia_root']}")
    print(f"Version: {config['catia_version']}")
    print(f"Code/Bin: {config['code_bin_path']}")
```

### Workspace Configuration

After `cade setup`, the workspace contains:

#### `.cade_workspace.json`
```json
{
  "workspace": "D:\\my_workspace",
  "catia_root": "D:\\CATIA\\B30",
  "catia_version": "B30",
  "cnext_exe": "D:\\CATIA\\B30\\win_b64\\code\\bin\\CNEXT.exe",
  "code_bin_path": "D:\\CATIA\\B30\\win_b64\\code\\bin",
  "copy_prereq": false
}
```

#### `setup_env.bat` (Windows)
```batch
@echo off
set "CATIA_ROOT=D:\CATIA\B30"
set "CATIA_VERSION=B30"
set "CNEXT_EXE=D:\CATIA\B30\win_b64\code\bin\CNEXT.exe"
set "WORKSPACE=D:\my_workspace"
set "CATIA_CODE_BIN=D:\CATIA\B30\win_b64\code\bin"
echo Workspace environment configured for CATIA B30.
```

## Integration with Prerequisites Manager

The detection system integrates with the Prerequisites Manager to automatically generate correct `AddPrereqComponent` paths:

### Before (Hardcoded)
```cpp
// IdentityCard.h
AddPrereqComponent("System", "Public", 
                   "C:\\Program Files\\Dassault Systemes\\B28\\intel_a\\code\\bin");
```

### After (Auto-detected)
```cpp
// IdentityCard.h (generated by CADE)
AddPrereqComponent("System", "Public", 
                   "D:\\CATIA\\B30\\win_b64\\code\\bin");  // Uses detected path
```

The `code_bin_path` from `.cade_workspace.json` is used when:
- Creating frameworks (`cade create framework`)
- Adding prerequisites (`cade prereq add`)
- Generating IdentityCard.h

## Benefits

### ✅ Portability
- **No hardcoded drives**: Works on any drive (C:, D:, E:, custom)
- **No hardcoded versions**: Supports B20-B99+, R2018+, future versions
- **No hardcoded paths**: Finds CATIA in any standard location

### ✅ Multi-Version Support
- Detects all installed CATIA versions
- User selects preferred version
- Different workspaces can use different versions

### ✅ Intelligent Selection
- Auto-selects newest version when only one found
- Sorts by version number (B30 > B28)
- Prioritizes B versions over R versions

### ✅ Validation
- Verifies CATIA installation structure
- Checks for CNEXT.exe existence
- Validates code/bin paths for AddPrereqComponent

## Migration Guide

### Old Workflow (Manual)
1. Open CATIA
2. Tools → Workspace Explorer
3. Right-click workspace → "Manage prerequisites"
4. Manually browse to CATIA installation (e.g., `C:\Program Files\Dassault Systemes\B28`)
5. Click OK

```bash
cd my_workspace
cade setup

```

   ```bash
   cade setup
   ```

   ```bash
   cade setup --show
   ```

## Technical Details

### CATIAInstallation Class

Represents a detected CATIA installation with:

- `root_path: Path` - Installation root directory
- `version: str` - Version string (e.g., "B28", "R2018")
- `release: str` - Full release name (e.g., "V5R28", "V5-6R2018")

Methods:
- `get_code_bin_path() -> Path` - Returns `<arch>/code/bin` path
- `to_dict() -> dict` - Serialize for JSON storage
- `from_dict(data) -> CATIAInstallation` - Deserialize

### CATIADetector Class

Core detection engine with:

Methods:
- `get_available_drives() -> List[str]` - Returns C-Z drives that exist
- `validate_catia_root(path) -> bool` - Validates CATIA structure
- `extract_version_info(dir_name) -> Tuple[str, str]` - Parses version
- `scan_directory(base_path) -> List[CATIAInstallation]` - Scans one path
- `scan_all() -> List[CATIAInstallation]` - Full system scan

### Detection Performance

- **Fast**: Typically 1-3 seconds on systems with 2-3 drives
- **Safe**: Handles permission errors gracefully
- **Cached**: Workspace config is saved, no re-scan needed

### Supported Architectures

Auto-detects the following CATIA architectures:
- `intel_a` (older versions)
- `win_b64` (64-bit Windows, most common)
- `win64` (alternative naming)
- `amd64_win64` (AMD64 architecture)

## Troubleshooting

### No CATIA installations found

**Symptoms**:
```
🔍 Scanning for CATIA installations...
❌ No CATIA installations found
```

**Solutions**:
1. Verify CATIA is installed
2. Check installation has `CNext/` directory
3. Manually specify path:
   ```bash
   cade setup --catia-root "X:\custom\path\to\CATIA"
   ```

### Multiple versions, wrong one selected

**Solution**: Use interactive mode to choose:
```bash
cade setup  # Select from list
```

Or specify explicitly:
```bash
cade setup --catia-root "D:\CATIA\B30"
```

### Permission errors during scan

**Symptoms**: Warnings about inaccessible directories

**Impact**: None if at least one CATIA found

**Solution**: Run with elevated permissions if needed (rare)
